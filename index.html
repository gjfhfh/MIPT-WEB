CineMood

Идея Сценарии Страницы API Стек

Каталог фильмов по настроению

Одностраничное приложение (SPA), которое помогает быстро подобрать фильм под конкретное настроение или ситуацию: «лёгкий вечер», «хочу пореветь», «интеллектуальное кино» и т.д.

Идея и проблема

Существующие сервисы (Кинопоиск, IMDb) дают огромные каталоги, но пользователю сложно быстро подобрать фильм под настроение. Приходится фильтровать по жанрам и рейтингам, тратить время на бесконечный скролл.

CineMood фокусируется на подборке фильмов по настроению и жизненным ситуациям. Пользователь выбирает настроение, а приложение показывает подходящие фильмы.

Чем лучше существующих решений?

  • Упор на «настроение», а не только жанры и рейтинг.
  • Быстрые пресеты: один клик — готовая подборка.
  • Списки «Избранное» и «Хочу посмотреть» с тегами настроения.

Ключевые пользовательские сценарии

1. Быстрый выбор фильма по настроению

  1. Пользователь открывает главную страницу.
  2. Выбирает настроение: «Лёгкий вечер» / «Комедия с друзьями» / «Хочу пореветь».
  3. Список фильмов обновляется и показывает подходящие варианты.
  4. Пользователь открывает карточку фильма и добавляет в «Хочу посмотреть».

2. Поиск конкретного фильма

  1. Пользователь вводит название фильма в строку поиска.
  2. Получает список результатов и выбирает нужный фильм.
  3. На странице фильма смотрит описание, рейтинг, теги настроения.
  4. Добавляет фильм в «Избранное».

3. Управление списком «Хочу посмотреть»

  1. Пользователь открывает страницу «Хочу посмотреть».
  2. Просматривает список запланированных фильмов.
  3. После просмотра меняет статус на «Просмотрено» и, при желании, оставляет заметку.

Структура страниц будущего SPA

Главная /

  • Блок выбора настроения (chip-кнопки).
  • Список популярных фильмов (карточки).
  • Поисковая строка по названию.

Страница фильма /movie/:id

  • Постер, описание, жанры, актёры, рейтинг.
  • Теги настроения.
  • Кнопки «Избранное» и «Хочу посмотреть».

Избранное /favorites

  • Список сохранённых фильмов.
  • Фильтрация по жанру и настроению.

О проекте /about

  • Описание идеи и стека технологий.
  • Ссылка на GitHub-репозиторий.

Черновое API (REST)

Фильмы

  • GET /api/movies/popular — список популярных фильмов.
  • GET /api/movies/{id} — детальная информация о фильме.
  • GET /api/movies/search?query=string — поиск по названию.
  • GET /api/movies?mood=chill&genre=comedy — фильтрация по настроению и жанру.

Избранное и «Хочу посмотреть»

  • GET /api/users/{userId}/favorites — список избранного.
  • POST /api/users/{userId}/favorites — добавить фильм в избранное.
  • DELETE /api/users/{userId}/favorites/{movieId} — удалить из избранного.
  • GET /api/users/{userId}/watchlist — список «Хочу посмотреть».
  • POST /api/users/{userId}/watchlist — добавить в список.
  • DELETE /api/users/{userId}/watchlist/{movieId} — убрать из списка.

План по реализации

Финальный проект будет реализован как SPA со следующим стеком:

  • TypeScript — типизация логики, компонентов и асинхронных операций.
  • React — функциональные компоненты и хуки.
  • Redux Toolkit — управление состоянием, createAsyncThunk для запросов.
  • React Router — навигация между страницами.
  • Бэкенд / внешнее API для фильмов (TMDB/OMDb или свой сервер).

На текущем этапе это только презентация идеи. К 20 декабря будет реализовано полноценное приложение со страницами, API и тестами.
